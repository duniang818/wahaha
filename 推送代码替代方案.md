# 推送代码替代方案

## 当前问题
网络连接问题导致无法直接推送代码：`Recv failure: Connection was reset`

## 推荐解决方案

### 方案1：使用GitHub Desktop（最简单）⭐

1. **下载GitHub Desktop**
   - https://desktop.github.com/
   - 下载并安装

2. **添加仓库**
   - 打开GitHub Desktop
   - File > Add Local Repository
   - 选择 `D:\githubio` 文件夹

3. **推送代码**
   - 登录GitHub账号
   - 点击 "Publish repository" 或 "Push origin"
   - 等待上传完成

### 方案2：GitHub网页上传

1. **访问仓库**
   - https://github.com/duniang818/duniang818.github.io

2. **上传文件**
   - 点击 "uploading an existing file"
   - 或者直接拖拽文件到仓库页面

3. **提交更改**
   - 填写提交信息：`Fix GitHub Pages configuration`
   - 点击 Commit changes

### 方案3：使用代理/VPN

如果有可用的代理或VPN：

1. 连接代理/VPN
2. 重试推送：
   ```bash
   git push origin main
   ```

### 方案4：使用SSH（如果已配置）

```bash
# 切换到SSH
git remote set-url origin git@github.com:duniang818/duniang818.github.io.git

# 推送
git push origin main
```

## 推送后的操作

无论使用哪种方法推送代码后：

1. **检查Actions**
   - 访问：https://github.com/duniang818/duniang818.github.io/actions
   - 查看 "Deploy to GitHub Pages" 工作流是否自动运行

2. **检查Pages设置**
   - Settings > Pages
   - 确保Source是 "GitHub Actions"

3. **等待部署**
   - 通常需要5-10分钟

4. **访问网站**
   - https://duniang818.github.io
