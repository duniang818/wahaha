飞博虾

工作台 · 发布台

在飞书写随笔、预览，再一键发到 GitHub 博客。无需自有公网——写作用飞书，托管用 GitHub。

WRITE

在飞书里写，就是最好的编辑与预览

台账

打开飞博虾台账

登记文档链接、栏目、定时与状态。

 云文档

新建一篇飞书文档

随笔直接写在 Docx 里，所见即所得。

 文档内

安装云文档小组件

开放平台开启「云文档小组件」后，在文档用 + / 插入飞博虾发布。
  1. 飞书云文档写好标题与正文
  2. 台账新增一行：粘贴文档链接，选「导航栏目」
  3. 状态选「待发博客」，或「一键操作」选「立即发布」
  4. 到「发布台」执行发布（本机或 GitHub Actions）

INBOX

稿件箱在飞书台账里

草稿 / 待发 / 已发 / 失败，全部用台账字段管理。这里不重复造列表——飞书表格本身就是稿件箱。

打开稿件台账 我的飞书云空间
建议视图：按「状态」分组；待发视图只显示「待发博客」。

PUBLISH

发布台（无自有公网）

飞书云端点不到你家电脑。发布有两条路，都借用 GitHub：

推荐 · 本机

本机发布台

npm run feiboxia:ship

拉飞书待发稿 → 写入 docs → push → Pages 自动部署。

备用 · GitHub

Actions 应用队列

本机先打包队列并 push，GitHub Actions 自动落到博客目录。

打开 Actions 页面 打开博客首页

「用 GitHub 触发」需要你在「设置」里保存 repo 与 PAT（workflow 权限）。Token 只存在本浏览器，不经过自建服务器。
