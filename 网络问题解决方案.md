# 网络连接问题解决方案

## 问题描述
无法连接到GitHub推送代码：`Recv failure: Connection was reset`

## 解决方案

### 方案1：使用GitHub Desktop（最推荐）⭐

**优点**：图形界面，操作简单，自动处理网络问题

1. **下载安装**
   - 访问：https://desktop.github.com/
   - 下载Windows版本并安装

2. **添加仓库**
   ```
   File > Add Local Repository > 选择 D:\githubio
   ```

3. **推送代码**
   - 登录GitHub账号
   - 点击 "Publish repository" 或 "Push origin"

### 方案2：网页上传（无需Git命令）

1. **创建仓库**
   - 访问：https://github.com/new
   - 仓库名：`duniang818.github.io`
   - 选择 Public
   - **不要**勾选任何初始化选项
   - 点击 Create repository

2. **上传文件**
   - 在仓库页面点击 "uploading an existing file"
   - 将 `D:\githubio` 文件夹中的所有文件拖拽上传
   - 在底部填写提交信息：`Initial commit`
   - 点击 Commit changes

3. **启用Pages**
   - Settings > Pages
   - Source: GitHub Actions
   - Save

### 方案3：配置代理（如果有代理服务器）

```bash
# 设置HTTP代理
git config --global http.proxy http://代理地址:端口
git config --global https.proxy https://代理地址:端口

# 推送代码
git push -u origin main

# 使用完后取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```

### 方案4：使用VPN

1. 连接VPN
2. 执行推送命令：
   ```bash
   git push -u origin main
   ```

### 方案5：使用移动热点

如果WiFi有问题，可以：
1. 使用手机热点
2. 连接后重试推送

### 方案6：修改DNS设置

有时DNS问题会导致连接失败：

1. **修改DNS为8.8.8.8**
   - 控制面板 > 网络和共享中心
   - 更改适配器设置
   - 右键网络连接 > 属性
   - Internet协议版本4(TCP/IPv4) > 属性
   - 使用下面的DNS服务器地址：8.8.8.8 和 8.8.4.4

2. **刷新DNS缓存**
   ```bash
   ipconfig /flushdns
   ```

## 验证连接

测试GitHub连接：
```bash
ping github.com
```

如果ping不通，说明网络确实有问题。

## 推荐操作流程

1. **首先尝试**：GitHub Desktop（最简单）
2. **如果不行**：网页上传（无需网络配置）
3. **最后尝试**：配置代理或VPN

## 上传后操作

无论使用哪种方法上传，都需要：

1. **启用GitHub Pages**
   - 访问：https://github.com/duniang818/duniang818.github.io/settings/pages
   - Source选择：**GitHub Actions**
   - 保存

2. **查看部署**
   - Actions页面：https://github.com/duniang818/duniang818.github.io/actions
   - 等待部署完成（约5-10分钟）

3. **访问网站**
   - https://duniang818.github.io

## 注意事项

- 确保仓库名是 `duniang818.github.io`（必须完全匹配）
- 确保仓库是Public（免费账户只能部署Public仓库）
- 确保 `.github/workflows/deploy.yml` 文件已上传

## 需要帮助？

如果所有方法都失败，可以：
1. 检查防火墙设置
2. 联系网络管理员
3. 使用其他网络环境（如手机热点）
